This guide shows you exactly how to hide/show specific toolbar elements and make the toolbar configurable.

1. üìù Minimal Toolbar (Only Basic Formatting)

Show only Bold, Italic, and Underline buttons:

const toolbarConfig = { preset: 'minimal' }; // Or be more specific: const toolbarConfig = { include: { buttons: ['bold', 'italic', 'underline'] } }; <WYSIWYGEditor toolbarConfig={toolbarConfig} />
Result: Only Bold (B), Italic (I), and Underline (U) buttons will appear in the toolbar.

2. üìö Hide Specific Buttons

Start with full toolbar but hide image and file upload buttons:

const toolbarConfig = { preset: 'full', exclude: { buttons: ['image', 'file', 'table'] } }; <WYSIWYGEditor toolbarConfig={toolbarConfig} />
Result: All buttons except image, file upload, and table insertion will be shown.

3. üéØ Category-Based Configuration

Show only formatting and structure categories:

const toolbarConfig = { include: { categories: ['formatting', 'structure'] } }; // Categories available: // - 'formatting': bold, italic, underline, strikethrough, subscript, superscript // - 'structure': h1, h2, h3, h4, h5, h6 // - 'lists': bulletList, numberedList, indent, outdent // - 'alignment': alignLeft, alignCenter, alignRight, alignJustify // - 'media': image, file, table // - 'links': link, unlink // - 'advanced': fontColor, backgroundColor, fontSize, etc. <WYSIWYGEditor toolbarConfig={toolbarConfig} />
Result: Only formatting buttons (Bold, Italic, etc.) and heading buttons (H1, H2, etc.) will be shown.

4. üö´ Hide Entire Categories

Start with full toolbar but hide media and advanced categories:

const toolbarConfig = { preset: 'full', exclude: { categories: ['media', 'advanced'] } }; <WYSIWYGEditor toolbarConfig={toolbarConfig} />
Result: All buttons except media (image, file, table) and advanced (colors, fonts, etc.) will be shown.

5. üìã Custom Button Selection

Pick exactly which buttons you want:

const toolbarConfig = { include: { buttons: [ 'bold', 'italic', 'underline', // Basic formatting 'h1', 'h2', 'h3', // Headings 'bulletList', 'numberedList', // Lists 'link', 'unlink', // Links 'undo', 'redo' // History ] } }; <WYSIWYGEditor toolbarConfig={toolbarConfig} />
Result: Only the specified buttons will appear in the toolbar.

6. üé® Custom Groups with Ordering

Organize buttons into custom groups:

const toolbarConfig = { include: { groups: [ { name: 'text-formatting', buttons: ['bold', 'italic', 'underline'] }, { name: 'headings', buttons: ['h1', 'h2', 'h3'] }, { name: 'actions', buttons: ['undo', 'redo', 'removeFormat'] } ] } }; <WYSIWYGEditor toolbarConfig={toolbarConfig} />
Result: Buttons will be organized into the specified groups in the given order.

7. üîß Real-World Example: Blog Editor

Configuration for a blog post editor:

const blogEditorConfig = { include: { categories: ['formatting', 'structure', 'lists', 'links'], buttons: ['image', 'undo', 'redo'] // Add specific buttons }, exclude: { buttons: ['subscript', 'superscript'] // Remove these from formatting } }; <WYSIWYGEditor toolbarConfig={blogEditorConfig} />
Result: Perfect for blog writing with formatting, headings, lists, links, and images, but without advanced features.

8. üìù Real-World Example: Comment Editor

Minimal configuration for user comments:

const commentEditorConfig = { include: { buttons: ['bold', 'italic', 'link'] } }; <WYSIWYGEditor toolbarConfig={commentEditorConfig} />
Result: Very simple toolbar with just bold, italic, and link buttons - perfect for comments.

9. üìß Real-World Example: Newsletter Editor

Configuration optimized for email newsletters:

const newsletterConfig = { include: { categories: ['formatting', 'structure', 'alignment'], buttons: ['link', 'image', 'undo', 'redo'] }, exclude: { buttons: ['subscript', 'superscript', 'h4', 'h5', 'h6'] } }; <WYSIWYGEditor toolbarConfig={newsletterConfig} />
Result: Perfect for email newsletters with formatting, headings (H1-H3), alignment, links, and images.

10. üé® Custom Button Groups with Ordering

Organize buttons into custom groups with visual separators:

const customGroupsConfig = { include: { groups: [ { name: 'text-formatting', buttons: ['bold', 'italic', 'underline'] }, { name: 'headings', buttons: ['h1', 'h2', 'h3'] }, { name: 'lists', buttons: ['bulletList', 'numberedList'] }, { name: 'actions', buttons: ['undo', 'redo', 'removeFormat'] } ] } }; <WYSIWYGEditor toolbarConfig={customGroupsConfig} />
Result: Buttons organized into logical groups with visual separators between each group.

11. üìö Documentation Editor

Perfect for technical documentation:

const documentationConfig = { include: { categories: ['formatting', 'structure', 'lists', 'links'], buttons: ['sourceCode', 'undo', 'redo'] }, exclude: { categories: ['media', 'advanced'] } }; <WYSIWYGEditor toolbarConfig={documentationConfig} />
Result: All formatting and structure tools, lists, links, and source code view - no media or advanced styling.

12. üéØ Using with Basic Toolbar Component

The basic Toolbar component also supports configuration:

import { Toolbar } from './components/Toolbar'; import { ToolbarConfigResolver } from './utils/toolbarConfigResolver'; const config = { include: { buttons: ['bold', 'italic', 'h1', 'h2'] } }; const resolvedConfig = ToolbarConfigResolver.resolve(config); <Toolbar onCommand={handleCommand} activeFormats={activeFormats} canUndo={canUndo} canRedo={canRedo} toolbarConfig={resolvedConfig} />
Result: Basic toolbar with only the specified buttons, maintaining backward compatibility.
